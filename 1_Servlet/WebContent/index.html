Hello, Servlet world!

서블릿이란, 웹 서비스를 위한 자바 클래스를 말하며, 자바를 사용해 웹을 만들기 위해 필요한 기술로 javax.sevlet.http.HttpServlet클래스를 상속 받는다.
다시말해 기존의 java파일에 웹 페이지 구현을 위한 html이 들어간 구조라고 볼 수 있다.
클라이언트의 요청을 처리하고 그 결과를 HTML을 사용하여 요청에 대한 응답으로 다시 클라이언트에게 전달하는 구현 규칙이 있다.
단, servlet에 작성된 HTML코드 변경 시 재 컴파일 해야 한다는 단점이 있다.

Servlet - 1 (GET)

GET 방식은 URL에 변수(데이터)를 포함시켜 요청하는 것으로 보안 유지가 불가능하다.
따라서 로그인, 회원가입 등의 경우는 GET방식으로 요청하는 것은 부적합함.
GET방식은 데이터를 HTTP Header에 포함하여 전송하는데 이 경우 보통 Body는 비어있는 상태로 전송이 됨
Header에는 들어갈 수 있는 데이터의 길이에 제한이 있으며 초과되는 경우 데이터가 절단된다.
Get방식은 캐싱(즐겨찾기, 북마크, 링크공유)가 가능하다.

Servlet - 2 (POST)

POST 방식은 URL에 변수(데이터)를 노출하지 않고 요청하는 것으로 보안 유지가 가능하다.
HTTP Body에 데이터를 포함하여 전송하므로 데이터 길이에 제한은 없지만, 대신 최대 요청 받는시간(Time Out)이 존재해 페이지 요청에 대해 기다리는 시간이 있음
POST 방식은 URL에 데이터가 노출이 되지 않으므로 캐싱 불가능.
